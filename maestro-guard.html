The Problem

AI Code Looks Right. Until It Doesn't.

LLMs are incredible at generating code that appears correct. But they consistently produce the same class of bugs — and manual review misses them.

🔗

Broken DOM References

getElementById('user-form') but the HTML has id="user-form " (trailing space). The AI hallucinated the reference — page silently breaks.

🪹

Empty Function Stubs

AI fills in function handleSubmit(e) { // TODO } — it looks like real logic, but the button does nothing. Shipped as "complete."

🔊

console.errors in Production

Debug console.error() calls left in the output. The AI used them during development and forgot to strip them. Your users see red in the console.

💥

Silent Syntax Errors

An unbalanced brace or a missing bracket. The browser swallows it, the page partially renders, and you chase a phantom bug for hours.

Deterministic Verification

The 4 Checks — No False Positives

Each check must score 100/100 to pass. No partial credit. No silent failures. No interpretation — just deterministic truth.

🔗

DOM References

Weight: 20  ·  dom_refs

Scans every getElementById, querySelector, and getElementsByClassName call and verifies the target actually exists in the HTML.

🪹

Empty Function Stubs

Weight: 25  ·  handlers

Detects empty function bodies, // TODO stubs, and casing mismatches between event handlers and their definitions.

🔊

console.errors in Production

Weight: 15  ·  console_errors

Flags any console.error() or console.warn() calls the AI left behind. These are always debug artifacts — never intentional in production code.

💥

Syntax Errors

Weight: 25  ·  js_syntax

Parses JS for unbalanced braces, brackets, and parens, plus broken string literals. Catches the silent breakers before they reach a browser.

Heuristic Review

5 Expert Perspectives. One Command.

Beyond deterministic checks, maestro-guard review analyzes your code from five distinct lenses — no API keys, no external services, just built-in heuristics.

🛡️

Security Auditor

Inline event handlers, eval(), innerHTML, CSP headers, localStorage exposure.

📐

Code Quality

Dead code, magic numbers, var usage, console.log, deep nesting, readability.

🎯

UX Reviewer

Viewport meta, alt text, semantic HTML, ARIA roles, focus styles, contrast.

📋

Completeness

Page title, meta description, charset, favicon, broken links, missing sections.

💰

Business Viability

CTAs, pricing, social proof, navigation structure, value proposition clarity.

CLI Usage

Two Commands. Total Control.

check for deterministic verification, review for heuristic analysis. Designed to fit into any workflow.

📋 maestro-guard check
# Single file
$ maestro-guard check index.html

# With spec fulfillment
$ maestro-guard check index.html --spec spec.md

# CI-friendly JSON output
$ maestro-guard check index.html --json

# Check all files in directory
$ maestro-guard check ./output/

# Verbose mode (show skipped checks)
$ maestro-guard check index.html --verbose
🔍 maestro-guard review
# Full 5-perspective review
$ maestro-guard review index.html

# Specific roles only
$ maestro-guard review index.html --roles security,ux

# Get structured fix suggestions
$ maestro-guard review index.html --improve

# CI integration
$ maestro-guard check ./output/index.html --json
