How It Works

One command. One spec. Verified output.

1
📝

Write a Spec

Document what you want built. Maestro decomposes it into tasks and builds each one using surgical edits — one change at a time, verified immediately.

2
🔍

Guardian Verifies

After every build phase, the Guardian runs 14 checks: JS syntax, DOM refs, handler definitions, console errors, spec fulfillment. Must score 100/100 to pass.

3
🚀

Ship Confidently

If any check fails, the circuit breaker stops the build. No partial passes. No silent failures. Either it passes 100% or it doesn't ship.

Key Features

What makes Maestro different from just running your AI tool and hoping for the best.

🛡️

14-Point Guardian Verification

Every build passes through mandatory checks: JS syntax, DOM reference integrity, handler definitions, console error detection, spec fulfillment, browser verification, and server validation. No gaps.

🧠

Multi-LLM Review System

5 expert reviewer roles — Security Auditor, Code Quality Analyst, UX Reviewer, Completeness Checker, Business Viability Reviewer — each using a different model via OpenRouter. Scores and issues from every perspective.

⚡

Circuit Breaker Protection

Prevents runaway builds from burning your API budget. Configurable max cost ($0.50 default), retry limits, cooldown periods, and automatic circuit state transitions. CLOSED → OPEN → HALF_OPEN.

🔧

Surgical Edit Engine

LLMs generate only old_string/new_string patch commands, not full files. Applied deterministically, verified immediately (page renders + no console errors), reverted automatically if verification fails.

📋

Spec Fulfillment Gates

Verifies every spec keyword appears in the output. If the spec says "dark theme with gold accents" and the output doesn't contain it, the Guardian fails the build. No silent omissions.

🔌

CI-Ready, Zero Web Service

JSON output, exit code 0/1. Pipe it anywhere. No database. No dashboard. No web service. Just a CLI tool that runs in your CI pipeline or on your laptop.

What Gets Checked

Every check must score 100/100. No partial credit. No silent failures.

20pts

Fulfillment

Spec keywords present in output. If it's in the spec, it must be in the build.

15pts

JS Syntax

Unbalanced braces, broken strings, syntax errors — per block, precisely located.

15pts

Handlers Defined

Function stubs are real, not empty. Casing matches between definition and usage.

8pts

No Console Errors

No .error(), .warn(), or debug logs left in production code.

8pts

Browser Verified

Headless render with zero console errors. Real page load, real verification.

5pts

Server Responding

Dev server starts, responds, and serves the correct file.

4pts

i18n Complete

All translations present. No missing language blocks.

2pts

DOM References

getElementById('x') actually matches id="x" in the HTML.
